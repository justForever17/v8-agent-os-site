Runtime-first Agent OS

让 Agent 记得住、扛得住，也能把计划带到明天早上。

很多 Agent 产品只适合演示，不适合长期工作。 V8 Agent OS 想做的是另一件更难也更有价值的事：让记忆留下来，让工具有纪律，让自动化继续跑，让 runtimes 在长任务、handoff、暂停和恢复里依然说同一种语言。

  • 重要记忆不会随着会话结束一起蒸发
  • Skills 与 MCP 定向暴露，不让 token 烧在无关工具上
  • 既能吃到 OpenClaw 生态，又能守住更稳的 runtime 主链

Runtime atlas

九个 runtimes，拼成一台遇到真工作也不散架的 Agent 机器。

V8 Agent OS 不相信一个大 prompt 能把所有事情都做好。记忆、自动化、安全、插件、桌面执行与网络协作，都应该有自己清晰的 runtime 边界。

CHAT RUNTIME
负责规划、委派、回答，并把每个 run 从第一句到最后一次 handoff 串成一条线。
MEMORY RUNTIME
负责偏好、图谱、知识召回，以及会在明天继续有用的长期记忆。
AUTOMATION RUNTIME
负责 cron、hooks、追踪循环，以及不会丢上下文的 Supervisor 心跳唤醒。
EXTENSIONS RUNTIME
把庞大的 MCP 和 skills 目录压成当前任务真正需要的那几样，不让 token 白白燃烧。
PLUGIN HOST RUNTIME
接住 OpenClaw 插件、IM 桥接和外部工具通道，又不让系统越接越乱。
SAFETY RUNTIME
负责审批、护栏、审计和高风险动作治理，但不把整套系统卡成假死。
COMPUTER USE RUNTIME
当任务真的需要一块屏幕时，在护栏和审计下接手桌面操作。
RPA RUNTIME
把重复、可复制的界面流程自动化到足够省心，也足够稳定。
NETWORK SUPERVISOR RUNTIME
发现 peers、唤醒远端 Supervisor，并把局域网和广域网里的多节点协作串成一张网。
安装

从 GitHub 拉取官方 bootstrap，然后干净地拉起 Admin 9528 和 Engine 9530。

按平台复制命令打开终端即可快速安装。当前公开安装入口已经统一收口到 justForever17/v8-agent-os 主仓，首装和重装都走同一条 canonical 路径。

安装后你会得到

你会从官方 monorepo 得到最新的 Engine 与 Admin、所需依赖、本地 env 文件，以及一套按统一默认端口启动起来的控制与执行平面：Admin 9528、Engine 9530。Web 端仍建议以独立 app 形态分发。

Windows
powershell -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/justForever17/v8-agent-os/main/bootstrap.ps1 | iex"
