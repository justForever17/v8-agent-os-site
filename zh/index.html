OpenClaw 对比

OpenClaw 值得尊重，因为它把生态广度这件事做得非常醒目。V8 真正想赢的，不是“也有这些”，而是把这些能力接进来后，仍然让任务少重讲、少噪音、可接管。

为什么更顺手

差别不在于部件更多，而在于任务一旦跨过演示时长，这些部件会不会开始互相打架。

真正让使用体验拉开差距的，是 scoped context、选择性工具暴露、可见的 workflow 状态、人类检查点，以及能慢慢长成可复用执行的屏幕自动化。

project 和 workspace scope、durable workflow projection 以及可恢复 run，让任务跨过 handoff、暂停和下一次会话后，仍然像同一件事。

安装

一条命令，把控制台和执行引擎一起拉起来。

从 justForever17/v8-agent-os 跑 bootstrap，你会得到 Admin 9528、Engine 9530，以及一套可以开始配置模型、记忆、工具和自动化的干净入口。

安装后你会得到

一个能看见并接管任务的控制台，一个准备好接模型、记忆、工具与自动化的执行平面，以及一套不用四处猜端口的默认启动方式。Web 端仍继续单独分发。

Windows
powershell -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/justForever17/v8-agent-os/main/bootstrap.ps1 | iex"
