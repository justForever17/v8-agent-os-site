Runtime-first agent OS

Build agents that remember, recover, and still know the plan tomorrow.

Most agent products shine for one demo and fade the moment real work becomes messy. V8 Agent OS is built for the jobs that spill across days: memory that keeps paying off, tools that stay disciplined, automation that keeps moving, and runtimes that recover instead of improvising excuses.

  • Long-term memory that does not disappear when the tab closes
  • MCP and skills routed on purpose instead of flooding the model
  • OpenClaw ecosystem reach on top of a steadier runtime spine

Runtime atlas

Nine runtimes. One machine that stays coherent when the job turns real.

V8 Agent OS does not pretend one giant prompt can do every job well. Each runtime exists so memory, automation, safety, plugins, desktop execution, and networking can stay strong without collapsing into noise.

CHAT RUNTIME
Plans, delegates, replies, and keeps every run coherent from the first instruction to the final handoff.
MEMORY RUNTIME
Keeps durable preferences, graph recall, and knowledge alive so useful context survives the next day.
AUTOMATION RUNTIME
Schedules hooks, cron jobs, follow-ups, and heartbeat wake-ups without losing the story of the work.
EXTENSIONS RUNTIME
Turns huge MCP and skill catalogs into the few capabilities that matter right now, without token panic.
PLUGIN HOST RUNTIME
Absorbs OpenClaw plugins, IM bridges, and external tool channels without letting the system turn feral.
SAFETY RUNTIME
Records approvals, guardrails, audits, and high-risk control without freezing the whole system.
COMPUTER USE RUNTIME
Takes the screen when the job really needs a desktop, but stays under guardrails and review.
RPA RUNTIME
Automates repeatable UI workflows so the dull work becomes reliably boring instead of fragile.
NETWORK SUPERVISOR RUNTIME
Finds peers, wakes remote supervisors, and coordinates multi-node work across LAN and WAN.
Install

Pull the official bootstrap from GitHub, then bring up Admin 9528 and Engine 9530 cleanly.

One command per platform. The public install entry now comes from the unified justForever17/v8-agent-os repository, so first install and reinstall follow the same canonical path.

What you get

You get the newest Engine and Admin from the official monorepo, the required dependencies, local env files, and a clean control-and-execution stack on the shared default ports: Admin 9528 and Engine 9530. The Web app is still distributed separately.

Windows
powershell -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/justForever17/v8-agent-os/main/bootstrap.ps1 | iex"
Open script
Linux
curl -fsSL https://raw.githubusercontent.com/justForever17/v8-agent-os/main/bootstrap.sh | bash
Open script
macOS
curl -fsSL https://raw.githubusercontent.com/justForever17/v8-agent-os/main/bootstrap.sh | bash
Open script
